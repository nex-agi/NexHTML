**高度管理规则（HEIGHT AWARENESS RULES）**

- **图片高度影响：** 每个图片元素会占用显著的纵向空间，
- **列间高度均衡：** 每列的内容高度应该大致相近，需参考图片高度信息分配剩下的文字内容的长度
- **空间利用目标：** 最大化每列空间使用率，避免溢出或留白

**关键规则：超大图片元素排除**

- **排除条件：** 若图片元素在 visual_heights_info 中的 `height_percentage > 50%`，必须排除
- **替代策略：**
  1. 从同类图片中选择高度 ≤50% 的替代项；
  2. 若无替代项，则转化为简洁的文字摘要；
  3. 确保逻辑连贯与叙事一致。
- **示例：**
  - 将结果表格改为要点摘要：“性能分析显示较基线提升 15%”。

**关键限制：**

**空间化规划要求**

**代码风格**
- 避免使用深色背景，浅色（白色、米色、浅灰色等等最佳）
- 可以适当使用一些css修饰
- 配色和样式以简约为主
- 使用标准的三栏布局结构，遵守如下形式：
  ```html
  <div class="poster">
    <div class="col left"> ... </div>
    <div class="col middle"> ... </div>
    <div class="col right"> ... </div>
  </div>
  ```
- 三栏容器 .poster 应具有固定的水平排列属性，例如：
  ```html
  .poster {
    display: flex;
    justify-content: space-between;
    align-items: flex-start;
    gap: 24px;
  }
  ```
- 中英文切换代码参考，en和中文切换按钮放成一个按钮吧：
  ```
  <div class="content-wrapper" data-lang="zh">
    <div class="content-zh">中文内容</div>
    <div class="content-en">English content</div>
  </div>


  javascriptfunction switchLang(lang) {
    const wrapper = document.querySelector('.content-wrapper');
    wrapper.setAttribute('data-lang', lang);
  }


  css.content-wrapper[data-lang="zh"] .content-en,
  .content-wrapper[data-lang="en"] .content-zh {
    display: none;
  }
  ```
- 每栏 .col 宽度保持一致（例如 width: 32%），并在竖直方向不能自由扩展。
- 避免使用自动换行或响应式 grid（如 display: grid 或 flex-wrap），确保三栏在标准桌面视口（宽度≥1200px）下一行展示。
- 各栏之间应有足够的水平间距（建议 20–40px），以保证几何检测的列间分离明显。
- 栏内可包含若干视觉模块（如 <section> 或 <div>），但不要嵌套额外的 .col 或 .poster。
- 页面整体容器的宽度可固定为 1200–1400px，并水平居中显示。
- 三个栏目的内容相近，不要有额外内容
