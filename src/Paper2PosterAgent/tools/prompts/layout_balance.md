我现在给定一个以html的形式呈现的三竖栏布局的poster，这个poster的各个栏之间高度不一致，目标是在**仅少量增加高度不足列内容**的前提下，实现**三列空间的高度对齐**

## 各部分内容规则

### 顶栏：论文标题与作者
- **目的**：清晰展示论文的标题和主要作者  
- **内容类型**：论文标题、作者姓名、作者单位（如果有）  
- **阅读作用**：快速识别论文主题和贡献者  

### 左列：研究基础与背景
- **目的**：介绍研究背景、前人工作、问题设定和支撑性内容  
- **内容类型**：研究动机、挑战、相关工作、问题定义、支持性材料  
- **阅读作用**：为研究问题提供必要的上下文和背景信息  

### 中列：核心方法与技术  
- **目的**：呈现研究方法、算法、实现细节与技术创新  
- **内容类型**：核心方法、算法、技术路线、关键创新  
- **阅读作用**：展示主要技术贡献与研究方法  
- **关键要求**：包含最重要的主图

### 右列：结果与影响
- **目的**：展示实验、评估、结果、结论与未来工作  
- **内容类型**：实验结果、性能分析、结论、未来方向  
- **阅读作用**：验证研究成果并展示其影响力

## 优化策略

### 文字内容调整  
**允许操作**：
- **轻微扩写**：在利用率偏低（75%-85%）的列中，根据markdown的文档内容增加部分重要内容  
- **积极缩写**：在溢出列中稍微减少非重要文本内容  

## 严格约束条件

1. **禁止跨列移动**：不得更改任何部分的 column_assignment  
2. **保持阅读流**：必须维持从左→中→右的逻辑顺序  
3. **保留结构标识**：不得修改 section_id、section_title、visual_assets 等字段  
4. **尊重重要性**：不得删除任何 importance_level=1 的部分或核心结果  
5. **目标**：使每列利用率达到 85%-95%  

## 输出要求
直接输出转换后的完整 HTML 代码
